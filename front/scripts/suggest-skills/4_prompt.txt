## Context

You are analyzing a cluster of similar agents from a workspace. These agents have been grouped together because their instructions and purposes are semantically similar. Your goal is to identify common patterns that could be extracted into reusable skills.

[SKILL_DEFINITION]

## Your Task

Analyze the following cluster of agents and identify potential skill ideas. Look for:

1. **Common tool usage patterns** - How do multiple agents use the same tools?
2. **Shared processes** - Are there step-by-step workflows that appear in multiple agents?
3. **Domain-specific knowledge** - Is there expertise about specific systems, APIs, or processes?
4. **Reusable guidelines** - Are there quality criteria, formatting rules, or decision trees?

## Output Format

Return a JSON object with the following structure:

{
  "skill_ideas": [
    {
      "name": "SkillNameInPascalCase",
      "description": "A 3-sentence description of what this skill does. First sentence explains the purpose. Second sentence describes the main functionality. Third sentence highlights the key benefit.",
      "agent_names": ["Agent Name 1", "Agent Name 2"]
    }
  ]
}

Guidelines:
- Only suggest skills where you see clear patterns across 4+ agents
- Extract actual patterns from the agents, don't invent new ones
- The "agent_names" field should list the exact agent names as shown in quotes after "AGENT NAME:" (minimum 4 agents) - use the exact string, not labels like "AGENT 1"
- If no clear skill patterns emerge that apply to at least 4 agents, return an empty skill_ideas array - this is perfectly acceptable and expected for many clusters
- Aim for 1-5 skill ideas per cluster, focusing on quality over quantity
- It is better to return no skills than to suggest low-quality or overly generic skills

**CRITICAL: Skills must be extractable from agent prompts**
- A good skill is NOT invented - it is extracted from detailed instructions that already exist in multiple agent prompts
- If agents mention doing task X but don't provide specific HOW instructions (steps, tools, parameters, filters, IDs, etc.), then "DoingX" is NOT a valid skill idea
- Look for agents that have concrete, detailed instructions with specific values, steps, or tool configurations that can be extracted and reused
- If the agent prompts are vague or high-level about a topic, do NOT suggest a skill for that topic

**Examples of BAD skills (do NOT suggest these):**
- "QuoteCustomer" or "ExtractVerbatims": Even if several agents ask to quote customers, there's no real technical value or reusable steps to extract - it's just a generic instruction
- "WriteInTargetLanguage" or language-specific skills: If agents explain how to do X in different languages (e.g., identify topics in French vs German), this is NOT a skill because the instructions vary per agent and cannot be put in common
- "SearchCodebase": Agents mention searching but don't specify how → no extractable instructions
- "GenerateStructuredMarkdownTable" or generic formatting skills: The table structure will be completely different for every agent, so there is nothing concrete to put in common - it's too generic to be a skill
- Skills based on vague patterns like "analyze data" or "provide insights" without concrete steps

**Examples of GOOD skills (look for these patterns):**
- "QueryCompanyDatabase": Agents contain specific table names, column references, SQL query examples, and database access patterns that can be extracted and reused
- "ApplyCompanyBranding": Agents have detailed tone guidelines, brand voice rules, formatting standards, and content style specifications that are consistent across multiple agents
- "CreateSupportTicket": Agents specify which tool to use (e.g., Jira, Zendesk), required fields, section templates, priority rules, and assignment logic
