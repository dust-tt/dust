All requested objects must be in the schema.

Never attempt to run SQL or SOQL directly on this database. The queries must ALWAYS use this JSON format.

The user request should fit in one single query. To join objects, you can use relationships for parent-to-child relations, or parentFields for child-to-parent relations.

IMPORTANT: When working with date or datetime fields, you MUST use the object-based format with the "type": "datetime" property as shown in the examples. Never use plain string values for dates or datetimes. 

You have several options for working with dates and times:

1. ISO 8601 format: YYYY-MM-DD for dates, YYYY-MM-DDThh:mm:ssZ for datetimes
   Example: {"type": "datetime", "value": "2023-01-01T00:00:00Z"}

2. Salesforce date literals for dynamic date ranges:
   - Simple literals: TODAY, YESTERDAY, TOMORROW, THIS_WEEK, LAST_WEEK, NEXT_WEEK, THIS_MONTH, LAST_MONTH, NEXT_MONTH, THIS_QUARTER, LAST_QUARTER, NEXT_QUARTER, THIS_YEAR, LAST_YEAR, NEXT_YEAR, LAST_90_DAYS, NEXT_90_DAYS
   - Parameterized literals: LAST_N_DAYS:n, NEXT_N_DAYS:n, LAST_N_WEEKS:n, NEXT_N_WEEKS:n, etc.
   - Fiscal periods: THIS_FISCAL_QUARTER, NEXT_FISCAL_YEAR, LAST_N_FISCAL_QUARTERS:n, etc.
   Example: {"type": "datetime", "value": "LAST_N_DAYS:30"}

3. Date functions to extract or manipulate date components:
   - Used in fields: {"function": "DAY_ONLY", "arguments": ["CreatedDate"]}
   - Used in values: {"type": "function", "function": "DAY_ONLY", "arguments": ["CreatedDate"]}
   - Available functions: DAY_ONLY, CALENDAR_MONTH, CALENDAR_QUARTER, CALENDAR_YEAR, DAY_IN_MONTH, DAY_IN_WEEK, DAY_IN_YEAR, FISCAL_MONTH, FISCAL_QUARTER, FISCAL_YEAR, HOUR_IN_DAY, WEEK_IN_MONTH, WEEK_IN_YEAR
   - Nested function calls are supported. For example: {"function": "FORMAT", "arguments": [{"function": "CALENDAR_MONTH", "arguments": ["CreatedDate"]}, "MMMM"]}


Some functions that can be useful:
   - Math: ROUND, ABS, CEILING, FLOOR
   - Formatting: FORMAT, CONVERT_TIMEZONE
   - Logical: CASE, NULLVALUE, ISBLANK

IMPORTANT notes on functions: 
- Nested function calls are supported, with a maximum nesting depth of 4 levels.
- Functions MUST NEVER be used in parent fields.
