# Dust iOS App

A native Swift/SwiftUI iOS app for Dust with WorkOS authentication.

## Requirements

- Xcode 16+
- iOS 17.4+
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) — `brew install xcodegen`
- [SwiftLint](https://github.com/realm/SwiftLint) — `brew install swiftlint`
- [SwiftFormat](https://github.com/nicklockwood/SwiftFormat) — `brew install swiftformat`
- [xcbeautify](https://github.com/cpisciotta/xcbeautify) (optional, prettier build logs) — `brew install xcbeautify`

The Xcode project (`Dust.xcodeproj`) is generated by XcodeGen from `project.yml` and is
git-ignored — never edit it by hand.

## Getting Started

```bash
cd x/adrsimon/mobile
make run            # generate the project, build the Dev config, launch on the simulator
```

To open the project in Xcode instead:

```bash
make generate
open Dust.xcodeproj
```

## Environments

The app builds in three flavors, selected by `ENV` (default `dev`):

| ENV       | Build config | Build type | API                     |
| --------- | ------------ | ---------- | ----------------------- |
| `dev`     | `Dev`        | debug      | `http://localhost:3000` |
| `prod`    | `Prod`       | debug      | `https://dust.tt`       |
| `release` | `Release`    | release    | `https://dust.tt`       |

`dev` points at a local `front` server, so it only works on the **simulator** (where
`localhost` is your Mac). For a physical device, point at your Mac's LAN IP instead.

All three flavors share the bundle id `com.dust.mobile`, so they can't be installed
side by side on the same device.

## Make targets

```bash
make help                 # list all targets
make build                # build for the simulator (ENV selects the config)
make run                  # build, boot the simulator, install, and launch
make test                 # run the test suite
make logs                 # stream the running app's logs from the simulator
make lint                 # SwiftLint
make lint-fix             # SwiftLint autocorrect
make format               # SwiftFormat (writes changes)
make format-check         # SwiftFormat in lint mode (no writes)
make clean                # remove build artifacts
make generate             # (re)generate Dust.xcodeproj from project.yml
```

Every env-specific task has two equivalent forms:

```bash
make run:prod             # same as: make run ENV=prod
make build:release        # same as: make build ENV=release
```

Override the simulator with `SIMULATOR`:

```bash
make run SIMULATOR='iPhone 16 Pro'
```

## Releasing to TestFlight

The app is signed with automatic signing against the Dust Apple Developer team
(`DEVELOPMENT_TEAM` is set in `project.yml`). The bundle id `com.dust.mobile` and the
App Store Connect app record already exist, so releasing is just archive → upload.

### One-time setup (per contributor)

1. Be a member of the team in [App Store Connect](https://appstoreconnect.apple.com)
   (Users and Access) with the **Developer** or **App Manager** role — an admin invites
   you by email.
2. Sign into Xcode with that Apple ID: **Xcode → Settings → Accounts → +**. Without this,
   signing can't provision and the archive fails.

That's it. On your first archive, signing provisions automatically (you may get a keychain
prompt to allow `codesign` to use the signing key).

### Cutting a release

First, bump the build number — it **must** be unique and higher than the last upload:

```
# project.yml: CURRENT_PROJECT_VERSION: N -> N+1
# Bump MARKETING_VERSION too for a user-facing version change (e.g. 0.1.0 -> 0.2.0).
```

Then archive and upload. The simplest path does both in Xcode, no extra tools:

```bash
make generate && open Dust.xcodeproj
# set the destination to "Any iOS Device", then Product -> Archive,
# then Distribute App -> App Store Connect -> Upload.
```

The build processes in App Store Connect (~5–15 min) and appears under the **TestFlight** tab.
Internal testers get it immediately; external testers get it after Apple's one-time Beta App
Review of the first build in a group.

## Architecture

```
Dust/
├── App/          # @main entry point, deep-link handling
├── Config/       # AppConfig: API base URL (per environment), endpoints, URL scheme
├── Models/       # Codable models (User, Conversation, Message, streaming events, …)
├── Services/     # API client, auth/PKCE, keychain, SSE streaming, transcription, uploads
├── ViewModels/   # Observable view models driving the UI
├── Views/        # SwiftUI views and components
└── Resources/    # Assets.xcassets (colors, app icon)
```
